ros有关的操作

搭建一个最简单的工作空间

ros工作空间的使用

第一步 创造工作空间并初始化
mkdir -p 自定义的空间名称/src
cd 自定义的空间名称
catkin_makebr

首先我们自己创建一个工作空间目录然后使用catkin_make编译

在我自己尝试时catkin_make经常出现错误Permission denied

解决方案如下 sudo chown $USER: -R /home/robot/catkin_ws

上面/home/robot/catkin_ws为自定义工作空间的目录，需要你根据自己的情况自行更改

Permission denied的错误都是权限不够，是需要改一下文件权限就可以了

第二步 进入src目录并添加依赖
cd src 
													catkin_create_pkg 自定义一个包名 roscpp rospy std_msgs
													//铸币的我在写这个的时候把msgs写成了msg（😃）,结果整个项目remake了一次(😃👋)
													//再补充：后面学了一下package.xml文件，这个地方应该不需要remake只需要把package.xml文件里的一些有关依赖的变量改一下就行
													//哦，还有CMakeLists.txt

roscpp是使用C++实现的库，rospy是使用python实现的库，std_msg则是标准消息库

使用ros时大多数时候用的就是这三个库

当你通过catkin_create_pkg创建了软件包之后src目录下会自动生成CMakeLists.txt文件与软件包名文件

软件包文件下会有include、src、CMakeLists.txt和package.xml文件

include中包含了需要使用的库

src中放置你所需要的CPP或Python源码

CMakeLists.txt中的内容是根据标准模板创建的，我们使用时需要稍微更改一下

找到136行的# add_executable(${PROJECT_NAME}_node src/learning_node.cpp)命令 这个命令在模板中是被注释掉的，我们需要取消他的注释

同时这行命令还需要我们的更改第一个参数为你项目名称，可以随意命名

第二个参数为你需要运行的cpp文件的名字

找到149行的 # targetlinklibraries(${PROJECTNAME}node # ${catkin_LIBRARIES} # ) 这三行命令都是被注释掉的，我们依旧取消他的注释

这个命令的第一个参数为项目名同上

第三步 在ros包中的src目录下放置需要使用的cpp代码
第四步 回到最初创建的工作空间目录下编译代码
cd 空间名
													catkin_make

报错没什么好说的，一般前面只要没问题那就不会报错

第五步 运行程序

在我们运行有关ros的程序时需要先在一个终端输入roscore命令

然后再开一个窗口，输入下列指令

cd 工作空间
													source ./devel/setup.bash
													rosrun 包名 节点名

包名就是你创建的软件包的名字，~~C++节点名是在你再CPP文件里初始化的节点名决定的~~

很有趣的发现，那个节点其实和CPP文件里的没关系，而是和你的CMakeLists文件有关，CPP节点即是你在CMakeLists所使用的工程名

你在CPP文件中使用的ros::init定义节点名称和CMakeLists的工程名是独立的

这两者之间的具体关系目前还不是很清楚，但已知的是他们俩是可以不同的，但通常是采用相同的命名，避免出现问题
